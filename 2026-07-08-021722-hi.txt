     SEO Agent Team: Content SEO + Kỹ thuật SEO

     Context

     User has 3 existing Claude Code skills (~/.claude/skills/):
     /seo-content-writer (writes article from a Google Sheet row → Google Doc),
     /compare-content-seo-competitor (crawls brand + competitor URLs → gap
     analysis), /duyet-bai-seo (reviews an article Doc against an outline Doc on
     4 criteria: keyword placement, outline coverage, spelling,
     images/captions).

     Goal: turn these into an automated 2-agent "SEO team" so that giving a
     keyword (+ some competitor URLs) produces a finished, reviewed, illustrated
     article with no manual steps in between. This requires:
     - 2 new subagents that each own 2 skills and know how to chain them.
     - 1 new skill, /toi-uu-bai-seo, that fixes the issues /duyet-bai-seo finds
     AND finds+inserts real images per H2 heading (via Google Images scraping —
     user explicitly chose scraping over the Custom Search API, accepting the
     fragility tradeoff).
     - A small extension to seo-content-writer's script so a bare keyword (no
     existing sheet row) can enter the pipeline.

     Decisions already confirmed with user (via AskUserQuestion):
     1. Keyword input → auto-append a new row to the existing Google Sheet, then
     run /seo-content-writer on it (no rewrite of that skill's core flow).
     2. Image search → scrape Google Images directly (not Custom Search API),
     accepting it's fragile/ToS-grey and may need revisiting later.

     Architecture

     User: "viết bài SEO cho từ khoá X, đối thủ: url1, url2"
             │
             ▼
     Main Claude session (orchestrator — no new file needed, just delegates in
     order)
             │
             ├─▶ Agent "content-seo"  (skills: compare-content-seo-competitor,
     seo-content-writer)
             │     1. /compare-content-seo-competitor on competitor URLs → gap
     report → derives outline
             │        (H2 candidates, LSI keywords/bigrams, missing formats)
             │     2. sheets_handler.py add-row <keyword> → gets row number
             │     3. /seo-content-writer <row>, using the derived outline as
     extra writing guidance
             │        on top of its fixed template (sapo/≤6 H2/kết luận rules
     unchanged)
             │     → returns Google Docs URL
             │
             └─▶ Agent "ky-thuat-seo"  (skills: duyet-bai-seo, toi-uu-bai-seo)
                   1. /duyet-bai-seo <outline-url> <article-url> <keyword> →
     review report
                   2. /toi-uu-bai-seo <article-url> <keyword>, using the review
     findings already
                      in context to decide concrete fixes:
                        - replace-text for spelling / keyword-placement fixes
                        - insert-after for missing outline sections
                        - insert-image for each H2 lacking an image (scrape →
     download → upload to
                          Drive → insert inline + italic caption)
                   → returns before/after summary

     The two agents are independent subagent files; I (the main assistant) chain
     them in sequence each time, passing the Docs URL from the first into the
     second. If the user gives only a bare keyword with no competitor URLs, I'll
     do a quick web search first to find 2-3 ranking competitor URLs before
     delegating to content-seo — this is my own orchestration behavior, not a
     file, since I'm the constant across the pipeline.

       - BƯỚC 0: bootstrap venv (same pattern as others)
       - BƯỚC 1: nhận article-url + keyword (+ the most recent /duyet-bai-seo

       - BƯỚC 2: áp dụng từng fix bằng scripts/optimizer.py replace-text

       - BƯỚC 3: với mỗi H2 heading chưa có ảnh — build a query from heading
     text + keyword, call scripts/image_search.py search "<query>", take first

     - scripts/optimizer.py — Docs+Drive write script, OAuth scopes documents +
     drive, same find_oauth_client() fallback chain as
     duyet-bai-seo/scripts/reader.py (own dir → duyet-bai-seo →
     seo-content-writer), own credentials/token.json. Commands:
       - replace-text <url> <old> <new> → batchUpdate with replaceAllText
       - insert-after <url> <anchor_text> <content> [--heading N] → fresh
     documents().get(), locate the paragraph whose text matches anchor_text
     (first match, warn on ambiguity), insert at its endIndex; if --heading N
     given, follow with updateParagraphStyle to set HEADING_N
       - insert-image <url> <anchor_text> <image_url> <caption> → download image
     bytes (requests, browser UA), upload to Drive (drive.files().create +
     public reader permission) to get a fetchable URI, locate anchor_text
     paragraph endIndex, batchUpdate insertInlineImage, re-fetch to get the
     fresh insertion point, second batchUpdate to insertText the italic caption.
     Two read-modify cycles (not one batch with manually-offset indices) to
     avoid index-math bugs — same "fresh read before each write" pattern already
     implicit in how the other skills' scripts are invoked repeatedly.
     - scripts/image_search.py — search "<query>": GET
     https://www.google.com/search?q=...&tbm=isch with a desktop User-Agent
     (same header style as compare.py's HEADERS), regex-extract direct image
     URLs from the raw HTML/embedded JSON, filter out gstatic.com thumbnails,
     dedupe, return top 5 as a JSON array to stdout. Clearly best-effort; empty
     result + stderr note if Google blocks/changes format.
     - install.sh, requirements.txt (google-api-python-client,
     google-auth-oauthlib, requests, beautifulsoup4, lxml) — copy the exact
